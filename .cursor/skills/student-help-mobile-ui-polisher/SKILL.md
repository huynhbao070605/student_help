# Student Help Mobile UI Polisher

Use this skill for cute mobile UI polish and UX improvements.

## Required Context

Before polishing UI, read:

- `docs/UI_STYLE_GUIDE.md`
- `docs/PRD.md`
- `docs/DECISIONS.md`
- `.cursor/rules/030-mobile-ui-style.mdc`

## Workflow

1. Start from the student journey and Android phone ergonomics.
2. Use warm pastel colors with clear contrast.
3. Prefer bottom tabs, cards, sheets, sticky actions, chips, and badges.
4. Keep text short, friendly, and Vietnamese-first where appropriate.
5. Use clear visual states for verified, pending, rejected, review, blocked, and consent-required flows.
6. Make empty states friendly and action-oriented.
7. Check small-screen layout for overflow and cramped touch targets.
8. Avoid desktop tables and web-like layouts.
9. Keep motion subtle and Android-friendly.
10. Update documentation if a reusable UI convention is introduced.

## Style Guardrails

- Cute and friendly, but still practical.
- No corporate SaaS feeling.
- No cluttered dashboards for student flows.
- No tiny tap targets.
- No one-off visual style that conflicts with the guide.
